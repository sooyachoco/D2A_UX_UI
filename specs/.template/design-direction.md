## 여백 시스템 (Spacing Scale)

> 4px 기반 그리드. 의도적 변주를 두어 시각적 리듬을 만든다.

| 토큰 | 값 | 용도 |
|---|---|---|
| space-1 | 4px | 인라인 요소 간 최소 간격 |
| space-2 | 8px | 밀접한 요소 간격 (라벨-입력 등) |
| space-3 | 12px | 컴포넌트 내부 패딩 (소) |
| space-4 | 16px | 컴포넌트 내부 패딩 (기본) |
| space-5 | 20px | 카드 내부 패딩 |
| space-6 | 24px | 관련 컴포넌트 그룹 간 간격 |
| space-8 | 32px | 섹션 내부 블록 간 간격 |
| space-10 | 40px | 콘텐츠 블록 간 간격 |
| space-12 | 48px | 소 섹션 간 간격 |
| space-16 | 64px | 섹션 간 간격 |
| space-20 | 80px | 대 섹션 간 간격 |
| space-24 | 96px | 히어로/주요 영역 여백 |
| space-30 | 120px | 페이지 최상위 여백 |

> 섹션 간 여백 ≥ 컴포넌트 내부 여백 × 2 (근접성 원칙).
> 콘텐츠 밀도에 따라 여백을 조절한다 — 모든 섹션에 동일 여백을 반복하지 않는다.

## 모션 가이드

> 정적 UI를 금지한다. 모든 인터랙티브 요소에 적절한 피드백을 제공한다.

### Duration

| 토큰 | 값 | 용도 |
|---|---|---|
| duration-fast | 150ms | 호버, 포커스, 토글 |
| duration-normal | 300ms | 패널 열기/닫기, 드롭다운 |
| duration-slow | 500ms | 페이지 전환, 히어로 진입 |
| duration-dramatic | 800ms | 첫 진입 애니메이션, 스플래시 |

### Easing

| 토큰 | 값 | 용도 |
|---|---|---|
| ease-out-expo | cubic-bezier(0.16, 1, 0.3, 1) | 요소 진입 (빠르게 시작, 부드럽게 감속) |
| ease-in-out | cubic-bezier(0.4, 0, 0.2, 1) | 위치 이동, 크기 변화 |
| ease-spring | cubic-bezier(0.34, 1.56, 0.64, 1) | 바운스 효과, 토스트 진입 |

### 진입/퇴장 패턴

| 패턴 | 설명 | 적용 |
|---|---|---|
| fade-in | opacity 0→1 | 모든 페이지 진입 기본 |
| slide-up | translateY(20px)→0 + fade | 스크롤 반응 섹션 등장 |
| scale-in | scale(0.95)→1 + fade | 모달, 팝오버 진입 |
| stagger | 자식 요소 순차 등장 (50~100ms 간격) | 카드 그리드, 목록 아이템 |
| slide-out-down | 0→translateY(10px) + fade-out | 토스트 퇴장 |

### 마이크로 인터랙션 기준

| 요소 | 호버 | 포커스 | 클릭/액티브 |
|---|---|---|---|
| 버튼 | 배경 밝기 + scale(1.02) + shadow 강화 | ring + border-color | scale(0.98) |
| 카드 | translateY(-2px) + shadow 강화 | ring | — |
| 입력 필드 | — | border-color + 미세 glow | — |
| 링크 | underline + 색상 변화 | outline | — |
| 아이콘 버튼 | 배경 원형 나타남 + 회전/스케일 | ring | scale(0.9) |

## 히어로/키비주얼 방향

> 히어로 영역은 페이지에서 가장 높은 디자인 밀도를 가진다.
> "중앙 정렬 제목 + 부제 + 버튼"만으로 구성하지 않는다.

### 선택된 히어로 패턴

- 히어로 레이아웃: {비대칭 분할 / 풀블리드 타이포 / 벤토 그리드 / 겹침 레이어 / 스플릿 스크린 / 인터랙티브 캔버스 / 카드 모자이크 / 타임라인 / 오프센터 앵커 / 풀스크린 비주얼+오버레이 / 세로 리듬 / 대각선 분할 / 기타}
- 배경 처리: {메시 그래디언트 / 2색 선형 / 패턴 오버레이 / 이미지+블러 / 솔리드+장식 도형}
- 장식 요소: {블러 오브(blob) / 도트 패턴 / 라인 / 기하학 도형 / 글로우 / 파티클}
- 모션: {진입 fade-in + slide / stagger 등장 / 배경 미세 움직임 / 호버 패럴랙스}
- 타이포 처리: {gradient text / 부분 color 강조 / 크기 대비(40px+ 제목)}

### 대시보드/관리자 첫 화면 대안

> 전통적 히어로가 어울리지 않는 경우의 "인상 영역" 방향.

- 환영 배너: {사용자명 포함 여부, 배경 처리}
- 핵심 메트릭: {카드 수, 아이콘 스타일, 숫자 강조 방식}
- 퀵 액션: {타일 수, 레이아웃}
- 장식: {그래디언트, 배경 패턴, 아이콘 색상}

### z-index 스케일 (GNB 적용 시)

> GNB 내부 z-index가 9,999,999이므로, 프로젝트의 모든 요소는 이보다 낮아야 한다.

| 토큰 | 값 | 용도 |
|---|---|---|
| z-content | 1 | 일반 콘텐츠 |
| z-sticky | 100 | 고정 헤더, 사이드바 |
| z-dropdown | 1000 | 드롭다운, 팝오버 |
| z-modal | 10000 | 모달, 다이얼로그 |
| z-toast | 50000 | 토스트, 노티피케이션 |
| (예약) GNB | 9,999,999 | 넥슨 GNB 영역 (외부 스크립트) |

### 레이아웃 영향

- 자체 헤더는 GNB **아래**에 위치한다
- `position: fixed/sticky` 헤더 사용 시 GNB 높이를 `top` 오프셋으로 고려한다
- 참조 문서: `.cursor/rules/nexon-gnb-guard.mdc`
