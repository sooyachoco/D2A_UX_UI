# 사전 확인 항목

> 구현 시작 전에 확인·준비가 필요한 항목입니다.
> spec.md의 "기능 의존성 분류"에서 🔴 외부 연동으로 분류된 시스템에 대해 작성한다.

## 차단 정책 (Real-first)

> 모든 외부 연동은 **실제 값 필수** (Real-first) 정책이 적용된다.
> Mock으로 우회하지 않고, 연동 단위별로 실제 연결 → 테스트 → 통과 후 다음 단계로 진행한다.

| 시스템 | 관련 Phase | 설명 |
|---|---|---|
| {시스템명} | Phase {N} | {사유} |

**정책:**
- 모든 환경(개발 포함)에서 실제 값 필수
- 미확보 시 해당 Phase 시작 차단
- 각 시스템은 값 입력 → 환경변수 설정 → 연결 테스트 통과(✅) 순서로 해결
- **모든 🔴 차단 항목은 `근거` 필드 필수** — 정책 문서(`refs/policies/{file}.md`) 인용 또는
  `사용자 입력`·`운영 정책` 같은 명시적 비-정책 출처 명시. 근거 없는 항목은
  `mcp__d2a-harness__check_prerequisites_grounded`가 `ungrounded` 차단으로 보고한다.
  케이스 배경: `docs/case-studies/step27-validation-gap.md` 4.7 장치 1.

**관련 Phase**: 해당 시스템이 처음 사용되는 Phase를 명시한다.
이 Phase 시작 시 게이트 검증이 발동한다.

## 비개발자 가이드

> 이 섹션은 개발 경험이 없는 분도 외부 연동 차단을 해결할 수 있도록 안내합니다.
> 개발이 "외부 연동 차단"으로 멈추었을 때, 이 가이드를 따라 해결하세요.

### 전체 흐름 이해하기

```
1. 우리 프로젝트가 외부 시스템(결제, 인증 등)과 연결해야 합니다
2. 연결하려면 접속 주소, 인증키 등이 필요합니다 (집 주소와 열쇠와 같음)
3. 이 정보가 없으면 개발이 멈춥니다 (차단 상태)
4. 담당 부서에 요청하여 정보를 받으면 개발을 계속할 수 있습니다
```

### 차단 해결 3단계

**1단계: 무엇이 필요한지 확인하기**
위 "차단 항목 요약" 테이블에서 🔴 항목을 확인합니다.
각 항목의 "해결 방법"과 "담당" 열을 참고하세요.

**2단계: 담당 부서에 요청하기**
아래 "요청 메시지 템플릿"을 복사하여 담당 부서에 보내세요.
요청 방법: 이메일, 슬랙 DM, 또는 NCSR(사내 요청 시스템)

**3단계: 값을 받으면 AI에게 전달하기**
값을 받으면 Cursor 채팅에 다음과 같이 입력하세요:
→ `"prerequisite 확보해줘"` (전체 항목을 하나씩 안내받기)
→ 또는 `"{항목명} 발급받았어"` (특정 항목만 처리하기)

### 요청 메시지 템플릿

### 자주 묻는 질문 (FAQ)

**Q: "외부 연동 차단"이 뭔가요?**
A: 우리 프로젝트가 다른 회사/팀의 시스템과 연결해야 하는데, 연결에 필요한 정보(주소, 비밀번호 등)가
아직 없어서 개발을 진행할 수 없는 상태입니다.

**Q: 왜 실제 값이 반드시 필요한가요?**
A: 이 프로젝트는 Real-first 정책을 사용합니다. 가짜 데이터(Mock)로 우회하지 않고,
처음부터 실제 외부 시스템에 연결하여 개발합니다. 이를 통해 연동 문제를 초기에 발견할 수 있습니다.

**Q: 환경변수가 뭔가요?**
A: 프로젝트가 외부 시스템에 접속할 때 필요한 설정값(주소, 비밀번호 등)을 모아둔 파일입니다.
`.env.local`이라는 파일에 저장되며, 이 파일은 내 컴퓨터에서만 사용됩니다.

**Q: 값을 받았는데 어디에 넣어야 하나요?**
A: 채팅에 `"prerequisite 확보해줘"` 라고 입력하면 AI가 하나씩 안내해 줍니다.
직접 파일을 수정할 필요 없이, AI의 안내를 따라가면 됩니다.

**Q: 담당자가 누구인지 모르겠어요.**
A: 위 "차단 항목 요약" 테이블의 "담당" 열을 확인하세요.
담당자 정보가 비어있으면 채팅에 `"담당자 찾아줘"` 라고 입력하세요.

**Q: NCSR이 뭔가요?**
A: Nexon Customer Service Request의 약자로, 사내 서비스 요청 시스템입니다.
다른 부서에 공식적으로 업무를 요청할 때 사용하는 온라인 양식입니다.
URL: https://ncsr.nexon.com

**Q: NXAS SSO 연동 시 도메인에 제약이 있나요?**
A: 네, NXAS SSO는 허용된 도메인에서만 로그인 콜백이 동작합니다.
`localhost`는 허용되지 않으므로 로컬 개발 시 hosts 파일 설정이 필요합니다.
상세 안내: `refs/policies/authentication.md`의 A-3a(허용 도메인), A-3b(설정 방법) 참조
또는 채팅에 `"NXAS 도메인 설정 안내해줘"` 라고 입력하세요.

**Q: 도메인이 아직 정해지지 않았는데 개발할 수 있나요?**
A: 가능합니다. hosts 파일에 임시 도메인(예: `dev-myservice.nexon.com`)을 설정하면
내 컴퓨터에서 해당 도메인으로 개발할 수 있습니다. 실제 도메인이 정해지면 교체하면 됩니다.
상세: `refs/policies/authentication.md` A-3b 참조

## 기타 외부 의존성

| 항목 | 담당 | 상태 | 비고 |
|---|---|---|---|
| {예: AWS 계정 발급} | {인프라팀} | ⬜ 미확인 | {소요 시간 등} |

> **인증 연동이 필요한 경우** (`refs/policies/authentication.md` 참조):
>
> ### 공통: DNS 도메인 등록 (NXAS / INSIGN 모두 해당)
>
> | 항목 | 담당 | NCSR | 비고 |
> |---|---|---|---|
> | DNS 도메인 등록 (스테이지/라이브) | 사내인프라팀 | [`https://ncsr.nexon.com/CSR/Write/23/551`](https://ncsr.nexon.com/CSR/Write/23/551) | `authentication.md` A-3c 참조 |
>
> ⚠️ **도메인 등록 정책** (중요):
> - `nexon.com`은 **대외 고객용 라이브 서비스 전용** — 개발/스테이지/내부 용도로 신청 시 **반려**
> - 개발 환경 도메인은 반드시 **`nxgd.io`** 사용 (신규 개발전용 도메인)
> - 백오피스/관리용 서비스는 **`mgt.nexon.com`** 사용
> - SSO 기술적 사유로 nexon.com이 필요하면 NCSR에서 **"nexon.com 예외처리"** 선택 가능
> - 로컬 개발은 DNS 등록 불필요 (hosts 파일 사용) — 서버 배포 시점에 신청
> - 긴급 문의: `system_dns@nexon.co.kr`
>
> ### 공통: 네트워크 ACL (접근 제어)
>
> | 항목 | 담당 | NCSR | 비고 |
> |---|---|---|---|
> | 네트워크 ACL 신청 | 사내인프라팀 | [`https://ncsr.nexon.com/CSR/Write/16/327`](https://ncsr.nexon.com/CSR/Write/16/327) | `authentication.md` A-3d 참조 |
>
> ⚠️ **ACL은 DNS 등록과 별도 NCSR** — DNS 등록 폼에 ACL 필드는 없음:
> - NXAS SSO 라이브 (`nxas.nexon.com`): **ACL 불필요** (AnyOpen)
> - NXAS SSO Dev/Stage (`dev-nxas`, `stage-nxas`): **ACL 필요** (Private IP)
> - INSIGN (`signin.nexon.com`, `public.api.nexon.com`): **ACL 불필요** (공개)
> - API Gateway 업스트림: IP 화이트리스트 시 **ACL 필요** (A-7 참조)
> - 로컬 개발 단계에서는 ACL 불필요 — 서버 배포 시점에 필요 여부 판단
>
> ### 사내 인증(NXAS SSO) 사용 시
>
> | 항목 | 담당 | 비고 |
> |---|---|---|
> | 서비스 도메인 확정 | 개발팀/인프라팀 | ⚠️ NXAS 허용 도메인만 가능 — `authentication.md` A-3a 참조 |
> | DNS 도메인 등록 | 사내인프라팀 | NCSR `https://ncsr.nexon.com/CSR/Write/23/551` — ⚠️ 개발용은 nxgd.io 필수 |
> | NXAS 클라이언트 등록 | 플랫폼실 | NCSR `https://ncsr.nexon.com/CSR/Write/23/161` |
> | 네트워크 ACL 신청 (Dev/Stage) | 사내인프라팀 | NCSR `https://ncsr.nexon.com/CSR/Write/16/327` — 라이브는 불필요 |
> | Callback URL / Logout URL 확정 | 개발팀 | HTTPS 필수 (라이브), 허용 도메인만 사용 가능 |
> | 로컬 개발 hosts 설정 | 개발팀 | `localhost` 불가 → hosts 파일 설정 필요 — `authentication.md` A-3b 참조 |
>
> ⚠️ **NXAS SSO 도메인 제약사항** (중요):
> NXAS SSO는 **Callback URL / Logout URL에 허용된 도메인만** 사용할 수 있습니다.
> `localhost`, `127.0.0.1` 등은 허용되지 않으므로 로컬 개발 시 hosts 파일 설정이 필요합니다.
> 상세: `refs/policies/authentication.md` A-3a(허용 목록), A-3b(로컬 개발 가이드) 참조
>
