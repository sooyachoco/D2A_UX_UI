### API Gateway 경유 시 로컬 개발 구성

> API Gateway를 사용하는 프로젝트에서 로컬 개발 환경을 구성할 때의 인프라 패턴.
> 상세 제약사항과 전략은 `authentication.md` A-9 참조.

| 계층 | 로컬 환경 | 스테이지/라이브 환경 |
|---|---|---|
| 프론트엔드 | `dev-{서비스}.nexon.com:3000` (hosts 설정) | `{서비스}.nexon.com` (EC2 SSR) |
| API 호출 (CSR) | 로컬 백엔드 직접 또는 프록시 | `public.api.nexon.com/{서비스}` (INFACE Gateway) |
| 토큰 검증 (SSR) | `private.api.nexon.com` (로컬에서 직접 호출 가능) | `private.api.nexon.com` (EC2에서 호출) |
| 백엔드 | `localhost:8000` | EC2 (Gateway NAT IP 허용 필요) |
