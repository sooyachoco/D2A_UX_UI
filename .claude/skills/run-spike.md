# Run Spike

기술 선택이 필요할 때 일회용 검증 스크립트를 실행하고 결과를 기록한다.

## 규칙

- spike 코드는 `specs/{NNN}/spikes/`에만 생성
- 결과는 반드시 `specs/{NNN}/decisions.md`에 기록
- spike 완료 후 코드는 삭제하거나 .gitignore 처리
- 본구현 코드에 spike 코드를 복사하지 않음

## Step 1: 후보 정리

```
🔶 기술 검증이 필요합니다: {주제}

**비교 후보**:
  A) {후보 1} — {특징}
  B) {후보 2} — {특징}

**비교 기준**: {정확도 / 속도 / 비용 / 설정 난이도 등}

이대로 비교할까요?
```

## Step 2: spike 스크립트 작성

`specs/{NNN}/spikes/spike-{주제}.py` 또는 `spike-{주제}.ts`를 생성한다.

스크립트 요구사항:
- 각 후보를 동일 조건으로 테스트
- 결과를 정량적으로 비교 가능한 형태로 출력

## Step 3: 실행 및 결과 정리

```
## Spike 결과: {주제}

| 후보 | {기준1} | {기준2} | 총평 |
|---|---|---|---|
| {후보A} | {결과} | {결과} | {한줄평} |
| {후보B} | {결과} | {결과} | {한줄평} |

**권장**: {후보X} — {이유}

이 결과로 진행할까요?
```

## Step 4: decisions.md 기록

```markdown
### DEC-{NNN}: {주제} 🟢 확정
- 날짜: {YYYY-MM-DD}
- 맥락: {왜 이 결정이 필요한지}
- 후보: {비교 대상}
- spike: specs/{NNN}/spikes/{파일명}
- 결과: {비교 결과 요약}
- 결정: {최종 선택 + 근거}
- 영향: {다른 부분에 미치는 영향}
```

## Step 5: 정리

- decisions.md 해당 항목을 ✅로 업데이트
